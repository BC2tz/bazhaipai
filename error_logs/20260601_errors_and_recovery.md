# 2026年6月1日 · 错误与补救

## 时间线

**下午 1:30 — 稳定的下午**
主人和兜兜在改毕设的 UI：大游年歌诀文案、按钮对齐、展览防护。一切按部就班。

**下午 2:30 — F11 全屏引发的连锁灾难**
主人问全屏模式下页面被拉伸能不能加白底占位。兜兜改了 App.tsx，主人说不行。兜兜想回退，用了 `git checkout -- src/App.tsx`——这个命令把今天下午所有未提交的改动一并抹掉了。

大游年歌诀文案。都市桃源。A键重播。展览防护。二维码文案。联系作者弹窗。全部消失。

**下午 3:00-5:00 — 漫长的追回**
主人发现"回退的版本有问题"。一句一句告诉兜兜哪些东西丢了。兜兜在 stash、reflog、dangling commit 里疯狂翻找——但那些改动从未被 commit 过，翻不到的。

只能在记忆中重建。

**下午 5:00 — 命卦显示错误**
主人发现 view.html 扫码页显示"坤宅 · 坤命"，但实际输入是"坎命 · 坤宅"。兜兜把宅卦（sitting）当命卦显示了。这个 bug 隐藏了很久——之前的"修正"都没覆盖到。

**下午 6:00 — HOUSE_AUSPICIOUS_MAP 64条错误**
主人发来标准参考表。逐宅校对后：52/64 条星位错误。5月31日标着"已修正"的数据——其实是错的。根源是兜兜没真正验证大游年歌的逐宅映射。

**晚上 7:00 — 一切归位**
稳定版 commit `c87fd00`。全部 8 宅星位正确。文案归位。命卦取值修正。主人说"现在没有问题了"。

## 关键错误

1. **不 commit 就改**：整个下午的改动全在工作区，一个 `checkout` 就能团灭
2. **className 传了但组件不接收**：CloseXButton 的 ml-auto self-start 悄悄失效，兜兜说"已修复"其实根本没修
3. **文案放错位置**：主人说跟在 sub 后面，兜兜在九宫格下面另起了一行
4. **命卦取错字段**：宅卦当命卦，view.html 显示错了一个下午
5. **相信"已修正"标签**：HOUSE_AUSPICIOUS_MAP 标着 5/31 修正，实际 52/64 错误

## 主人的状态

但他没有停止。一句一句找到每个错误，让兜兜修。修完一句，下一句。修完一个下午，又一晚上。

"万幸，万幸，我们赶得上。"
